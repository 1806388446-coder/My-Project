# 阿里云动态回忆网站设计

## 背景

当前网站是纯静态页面，留言、纪念日、主题和上传照片都存放在访问者自己的浏览器 `localStorage` 中。上线后每个访问者只能看到自己的本地数据，无法共享留言和照片。

目标是把网站升级为动态网站：所有访问者都能打开同一个公网链接查看内容；被授权的人可以留言、上传照片和管理回忆内容；站点优先使用阿里云，避免购买长期运行的服务器。

## 目标

- 使用阿里云部署，支付和资源管理集中在阿里云账号内。
- 访问者无需登录即可查看公开内容。
- 管理员可以控制谁能编辑。
- 编辑者第一次授权后，在同一浏览器长期保持可编辑状态，除非主动退出或权限被撤销。
- 留言和照片对所有访问者同步可见。
- 上传照片时自动压缩，降低 OSS 存储和下行流量成本。
- 第一版不引入手机号、邮箱验证码或复杂账号体系。

## 非目标

- 不做公开注册。
- 不做社交平台式关注、点赞、评论审核流。
- 不购买 ECS 或轻量服务器长期跑后端。
- 不在第一版做复杂多角色后台，只提供足够管理编辑权限的轻量面板。

## 推荐架构

```text
浏览器
  ↓
OSS 静态网站托管：index.html / styles.css / app.js / assets
  ↓
函数计算 FC：API 接口
  ↓
TableStore：留言、瞬间、编辑者、邀请码、站点配置
OSS：上传照片文件
```

### 阿里云服务

- OSS 静态网站托管：放页面文件和项目内置资源。
- OSS 对象存储：放用户上传后的照片。
- 函数计算 FC：提供登录、读取、创建、编辑、删除、上传等 API。
- TableStore：存结构化数据，包括留言、瞬间、照片元数据、邀请码、编辑者授权和站点配置。

## 权限模型

### 访客

- 可以查看留言、瞬间和照片。
- 不能新增、编辑、删除内容。
- 看不到编辑按钮和管理入口，或者看到后进入时仍会被后端拒绝。

### 编辑者

- 通过管理员发放的邀请码激活。
- 可以新增留言。
- 可以新增瞬间并上传照片。
- 可以编辑或删除自己创建的内容。
- 在同一浏览器长期保持编辑状态。
- 可以手动退出编辑模式。

### 管理员

- 使用管理员密码登录。
- 可以生成、查看、禁用邀请码。
- 可以查看和撤销编辑者授权。
- 可以编辑或删除任何留言和瞬间。
- 可以修改云端站点配置，例如纪念日、TA 的称呼。

## 邀请码流程

1. 管理员输入管理员密码进入管理模式。
2. 管理员生成邀请码，并设置一个显示名称，例如“珊珊”。
3. 管理员把邀请码发给指定的人。
4. 对方在页面输入邀请码。
5. 后端校验邀请码有效后，创建编辑者身份并返回长期令牌。
6. 浏览器把长期令牌保存在本地。
7. 之后访问页面时，前端带上令牌请求后端校验；有效则自动进入编辑模式。
8. 管理员撤销编辑者后，该令牌下次请求时失效。

邀请码建议一次性使用。若需要重新授权，管理员生成新邀请码。

## 数据模型

### SiteConfig

- `id`: 固定为 `main`
- `partnerName`: TA 的称呼
- `anniversaryDate`: 纪念日起始日
- `updatedAt`: 更新时间

主题偏好继续保存在每个访问者浏览器中，不上云。

### Note

- `id`: 唯一 ID
- `authorId`: 创建者 ID
- `authorName`: 署名
- `text`: 留言内容
- `color`: 信纸颜色
- `createdAt`: 创建时间
- `updatedAt`: 更新时间

### Moment

- `id`: 唯一 ID
- `authorId`: 创建者 ID
- `authorName`: 创建者显示名
- `icon`: 类别图标
- `tag`: 类别名称
- `content`: 瞬间描述
- `date`: 瞬间日期
- `images`: 照片对象数组
- `createdAt`: 创建时间
- `updatedAt`: 更新时间

照片对象包含：

- `key`: OSS 对象路径
- `url`: 可访问 URL 或签名访问 URL
- `width`: 压缩后宽度
- `height`: 压缩后高度
- `size`: 压缩后字节数

### Editor

- `id`: 编辑者 ID
- `displayName`: 显示名称
- `status`: `active` 或 `revoked`
- `createdAt`: 创建时间
- `revokedAt`: 撤销时间，可为空

### InviteCode

- `codeHash`: 邀请码哈希，不保存明文邀请码
- `displayName`: 邀请码对应的显示名称
- `status`: `unused`、`used`、`revoked`
- `createdAt`: 创建时间
- `usedAt`: 使用时间，可为空
- `editorId`: 使用后绑定的编辑者 ID，可为空

## API 设计

### 公开接口

- `GET /config`: 读取云端纪念日和称呼。
- `GET /notes`: 读取留言列表。
- `GET /moments`: 读取瞬间和照片列表。

### 授权接口

- `POST /auth/admin/login`: 管理员密码登录，返回管理员令牌。
- `POST /auth/invite/claim`: 使用邀请码激活编辑者，返回编辑者令牌。
- `GET /auth/me`: 校验当前令牌是否有效，返回角色和显示名称。
- `POST /auth/logout`: 前端清除本地令牌即可，后端可选记录。

### 编辑者接口

- `POST /notes`: 新增留言。
- `PATCH /notes/:id`: 编辑自己的留言。
- `DELETE /notes/:id`: 删除自己的留言。
- `POST /moments`: 新增瞬间。
- `PATCH /moments/:id`: 编辑自己的瞬间。
- `DELETE /moments/:id`: 删除自己的瞬间。
- `POST /uploads/photos`: 上传照片，返回 OSS 照片对象信息。

### 管理员接口

- `PATCH /config`: 修改纪念日和称呼。
- `POST /admin/invites`: 生成邀请码。
- `GET /admin/invites`: 查看邀请码状态。
- `POST /admin/invites/:id/revoke`: 禁用邀请码。
- `GET /admin/editors`: 查看编辑者。
- `POST /admin/editors/:id/revoke`: 撤销编辑者。
- 管理员可调用所有编辑和删除接口，不受“只能操作自己内容”的限制。

## 前端改造

现有页面结构保留，主要替换数据来源：

- 启动时从云端读取 `config`、`notes`、`moments`。
- 留言板从 `localStorage` 改为 API 读写。
- 瞬间照片不再保存为 base64 字符串，而是上传到 OSS 后保存 URL。
- 增加“进入编辑模式”入口。
- 编辑者登录后显示新增、编辑、删除按钮。
- 管理员登录后显示轻量管理面板。
- 增加“退出编辑模式”按钮，清除本地令牌。
- 网络失败时保留已有页面内容，并展示简短错误提示。

## 上传约束

- 前端上传前压缩图片。
- 单张照片压缩后限制为 1 MB 以内。
- 单次最多上传 9 张。
- 支持常见图片格式。
- 压缩后仍超过限制时提示更换图片。
- OSS 路径按日期和随机 ID 存放，例如 `uploads/2026/06/<id>.jpg`。

## 安全设计

- 管理员密码不写死在前端代码中。
- 邀请码不以明文保存到数据库，保存哈希。
- 所有写接口都由函数计算后端校验令牌。
- 前端隐藏按钮只作为体验优化，不作为权限依据。
- CORS 只允许站点域名访问 API。
- 上传接口校验文件类型和大小。
- 删除 OSS 照片时，同时删除 TableStore 中的记录或标记内容已删除。

## 成本控制

- 不购买 ECS。
- 函数计算按调用付费。
- TableStore 存储和读写量很小。
- 主要成本来自 OSS 存储和照片下行流量。
- 通过前端压缩和单张大小限制控制费用。
- 建议阿里云设置月度预算告警，例如 20 元或 50 元。

## 域名、HTTPS 与备案

第一版支持两种访问方式：

- 快速上线：先使用 OSS 静态网站托管提供的访问地址，尽快获得可分享链接。
- 正式链接：注册自定义域名，绑定到 OSS 静态网站和函数计算 API，并配置 HTTPS。

推荐路径：

1. 在阿里云中国站注册域名，并完成域名实名认证。
2. 若使用中国香港或新加坡地域承载网站，先不做中国内地 ICP 备案也可快速上线。
3. 若未来把网站解析到中国内地地域的云资源，或使用中国内地接入服务承载网站，应按阿里云备案要求完成 ICP 备案。
4. 域名绑定到 OSS 后，通过阿里云 DNS 添加 CNAME 解析。
5. 为自定义域名配置 HTTPS 证书，最终使用 `https://<domain>` 访问。

建议域名结构：

- `www.<domain>` 或根域名：访问回忆网站。
- `api.<domain>`：访问函数计算 API。
- `static.<domain>` 或 `img.<domain>`：可选，用于照片资源；第一版可以先不用单独拆分。

如果不想先买域名，第一版仍可使用 OSS/函数计算提供的默认访问地址完成动态功能验证。

## 错误处理

- API 读取失败：显示当前静态默认内容或上一次加载的内容，并提示稍后重试。
- 登录失败：提示密码或邀请码不正确。
- 令牌失效：自动退出编辑模式，提示重新获取授权。
- 上传失败：保留表单文字和已选择的本地预览，允许重试。
- 删除失败：恢复被删除项的显示状态。

## 测试与验收

- 未授权访客只能查看，无法通过界面新增、编辑或删除。
- 未授权请求直接调用写接口会被拒绝。
- 管理员可以生成邀请码。
- 邀请码可被使用一次，使用后不能再次激活新人。
- 编辑者激活后刷新页面仍保持编辑模式。
- 编辑者手动退出后恢复访客模式。
- 管理员撤销编辑者后，该编辑者后续写操作失败。
- 编辑者只能修改或删除自己创建的内容。
- 管理员可以删除任何内容。
- A 设备上传留言或照片后，B 设备刷新能看到。
- 上传照片不会再写入 `localStorage` base64。
- 超大图片会被压缩或拒绝。

## 第一版实施边界

第一版实现共享留言、共享瞬间、照片上传、编辑者邀请码、管理员管理和云端纪念日配置。保留当前视觉设计和交互风格，不做大规模 UI 重做。
